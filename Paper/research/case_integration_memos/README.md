# SCOTUS Case Integration Memos

**Generated:** 2026-04-23 | **Plan:** `scotus-case-integration-read_c0ca9b63`

Each memo in this directory is a structured gap analysis for one Tier-1 SCOTUS case. Memos are read-only research artifacts — they **propose** integration snippets but do **not** edit the manuscript or `.bib` file. Integration is gated on explicit user approval.

---

## Memo Format

```
{slug}.md         ← Integration memo
{slug}_context.md ← Auto-generated by extract_case_context.py (raw tex paragraphs)
SUMMARY.md        ← Phase-4 aggregate: snippets-per-chapter + escalation decision
```

### Memo Structure (per case)

| Section | Contents |
|---------|----------|
| §1 What the book says now | Auto-extracted tex paragraphs keyed by cite key + party regex |
| §2 What the opinion actually holds | Curated summary with exact quotes + page refs |
| §3 Gap diagnosis | Checklist: missing quotes, mischaracterizations, missing doctrines, orphan opportunities |
| §4 Ready-to-paste LaTeX | `@case{}` bib entries, `\footnote{}` additions, body paragraphs — tagged by chapter |

## Tooling

- **`Paper/scripts/extract_case_context.py`** — Extracts every `.tex` paragraph mentioning a case (by cite key + party-name regex). Run before writing `§1 What the book says now`.
  ```bash
  python3 Paper/scripts/extract_case_context.py <slug_or_cite_key>
  ```
- **`Paper/scripts/audit_scotus_corpus.py`** — Build-time linter. After Phase 4, also warns if a Tier-1 case is missing a memo.
