## Challenges Encountered

1. **Tuple vs string env_lines**: The `env_lines` list was used in both single-label and multi-label paths. Changing to `(int, str)` tuples required updating `full_content = "\n".join(r for _, r in env_lines)` in the single-label branch to unpack correctly.
2. **`generate_readme.py` format code error**: `f"| {r.get('seq','?'):02d}"` fails when `seq` is a string (YAML parses integers, but the default fallback `'?'` is a string). Fixed with `.zfill(2)` after `str()`.
3. **Case study scanner scope**: Searching 200 lines after each equation risks capturing patterns belonging to a subsequent equation. Bounded the search at `min(next_eq_line, eq_line+200)` to prevent cross-contamination.
4. **eq:26 reclassification rationale**: The comment asked to reclassify or justify. Since Mapping Police Violence provides killings-per-capita by racial group, eq:26 genuinely supports quantitative estimation, justifying `type=quantitative, tier=2` over `ordinal, tier=3`.
