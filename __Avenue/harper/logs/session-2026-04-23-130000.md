## Challenges Encountered

1. **Pre-existing nbformat validation error**: The first three parallel execution attempts all failed with `nbformat.validator.NotebookValidationError: 'name' is a required property`. This is a pre-existing issue affecting all notebooks in the project — older cell outputs omit the `name` field required by newer nbformat. Fixed with a patch script before re-running.

2. **`display_data` outputs also fail validation**: After fixing `stream` outputs, the second run revealed that `display_data` outputs also need a `metadata: {}` field. Required a second pass of the patch script.

3. **`eq_funding_propval_feedback.ipynb` uses named axes (ax1–ax6), not an array**: Unlike the other notebooks that use `axes[i]` or `axes.flat`, this notebook uses individual variables. The panel-saving loop explicitly lists `[ax1, ax2, ax3, ax4, ax5, ax6]`.

4. **`spectral_laplace.ipynb` uses `axes.flat` with named years**: The 4 panels correspond to specific shock years (1865, 1964, 2008, 2020). Panel filenames include the year for clarity (`_panel_a_1865.pdf`, etc.).

5. **PDF panel files gitignored**: The `Paper/figures/spectral/` directory's PDF files are in `.gitignore`. They are generated locally but cannot be committed. This is expected behavior — PDFs are regenerated by running the notebooks.
