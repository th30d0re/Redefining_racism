## Challenges Encountered

1. **`chapter_max` seeded from empty registry map**: After the previous full migration, all registry files had `label == new_label`, so Phase A found 0 pairs. The auto-mapping function initially computed `chapter_max` only from Phase A's output — getting seq numbers 1, 2, 3... for each chapter instead of 19, 20... The fix was to seed `chapter_max` from ALL already-hybrid labels present in the manuscript (passed as `all_hybrid_labels: set[str]`), not just from the registry pairs.

2. **Chapter counting by line vs. character position**: The initial chapter map used line-based scanning but the manuscript is read as one string. Switched to character-position binary search, which is cleaner and correctly handles multi-line `\label{}` contexts.

3. **`parseval` ref count 3 in dry-run vs. 2 in apply**: One occurrence of `eq:parseval` in a figure caption was within text that was already modified by a prior substitution (dry-run counts against frozen original text; apply counts against modified text). Not a bug.

## Next Ideas (6 Ideas)

1. **Run coverage check as a standalone CI linter**: Extract `validate_coverage` into a separate `check_labels.py` script that can run on every PR/commit to catch any future non-hybrid label being introduced.

2. **Generate registry entries for the 6 auto-labeled equations**: Write minimal `.md` files for `eq:8.19-laplace-transfer`, etc., in `Paper/empirical_validations/` so they appear in registry-driven workflows (the current auto-generation only migrates the manuscript; no registry file is created).

3. **Migrate prose `eq:N` references**: The manuscript contains a handful of plain-text `eq:74`, `eq:73` style prose references (e.g., "This is eq:74 confirmed") that are outside the four LaTeX contexts handled by the script. A follow-up pass could convert or annotate these.

4. **Add `label` uniqueness assertion**: The script currently deduplicates label extraction but doesn't warn if the same label appears multiple times in the manuscript. A uniqueness check would catch accidental copy-paste duplication.

5. **Semantic descriptor improvement**: The current auto-descriptor is just a slugified version of the old label name (e.g., `eq:45b` → `45b`). A smarter version could extract the nearest `\paragraph` or `\subsection` title as the descriptor.

6. **Lock the hybrid label pattern in `.editorconfig` or pre-commit**: A pre-commit hook that runs `check_labels.py` and rejects any `\label{eq:...}` not matching `^eq:\d+\.\d+` would prevent future drift without manual auditing.
