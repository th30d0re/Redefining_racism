## What Was Wrong / What Was Requested

Three verification comments were filed after a thorough code review. Each identified a gap between the ticket acceptance criteria and the implemented state:

1. **Abstract/frontmatter not updated** (`Paper/Redefining_Racism.tex`): The ticket required both the genre-paragraph update *and* an abstract/frontmatter update to reflect the new empirical validation apparatus. Only the genre paragraph (`\paragraph{A note on formalism.}`) had been updated; the Preface (the document's functional abstract) contained no mention of the confidence-tier scheme, the 145 anchor cases, or `\ref{app:empirical_index}`.

2. **`make index` uses system Python** (`Makefile`): `pdf: index ...` was correctly wired, but the `index` target called `python3 Paper/scripts/generate_index.py` directly — bypassing the project's `.venv` virtual environment. Because `generate_index.py` hard-exits if `pyyaml` is unavailable, this silently breaks `make pdf` on any machine where PyYAML is not installed system-wide.

3. **LaTeX label keys escaped before `\pageref{}`** (`Paper/scripts/generate_index.py`): `_case_study_location_cell()` called `_tex_escape(s)` on the label token before interpolating it into `\pageref{...}`. This converts underscores to `\_`, which does not match the original `\label{...}` key and produces unresolved cross-references at compile time.

## How I Fixed It / What I Did

### Fix 1 — Preface empirical-upgrade paragraph (`Paper/Redefining_Racism.tex`)

Added a clearly marked `\textbf{Empirical validation.}` paragraph immediately after the final Preface paragraph. The new paragraph:
- Names the three confidence tiers (Tier 1/2/3) in the same language used throughout the body
- States the 145-anchor-case calibration scope
- References `Appendix~\ref{app:empirical_index}` for the per-equation index
- Cross-links to the Empirical Methodology chapter via `p.~\pageref{ch:empirical_methodology}`

### Fix 2 — `index` target uses venv Python (`Makefile`)

Changed:
```makefile
# Before
index:
    @if [ -f Paper/scripts/generate_index.py ]; then \
      python3 Paper/scripts/generate_index.py; \
```
to:
```makefile
# After
index: venv
    @if [ -f Paper/scripts/generate_index.py ]; then \
      $(VENV_PYTHON) Paper/scripts/generate_index.py; \
```
The `venv` prerequisite ensures the virtual environment is bootstrapped before `generate_index.py` is invoked. `$(VENV_PYTHON)` resolves to `.venv/bin/python3`, which has PyYAML (and all other `requirements.txt` packages) reliably available.

### Fix 3 — Raw label keys in `\pageref{}` (`Paper/scripts/generate_index.py`)

Introduced a `_validate_label_key(s)` helper that:
- Strips whitespace
- Validates the key matches `^[A-Za-z][A-Za-z0-9:_\-\.]*$` (the character set LaTeX allows in cross-reference keys)
- Returns the raw string if valid, `None` otherwise

Updated `_case_study_location_cell()` to:
- Call `_validate_label_key()` instead of `_tex_escape()` when the location string is identified as a LaTeX label
- Emit `p.~\pageref{raw_key}` with the unescaped key
- Fall back to a `\textit{(invalid label)}` display cell (rather than silently producing broken LaTeX) when the key contains characters outside the safe set

Display text cells (chapter numbers, line numbers) continue to go through `_tex_escape()` as before.

## Challenges Encountered

1. **Locating the "abstract" region**: The document uses a `\chapter*{Preface}` block rather than a traditional `\begin{abstract}` environment, so the comment's "frontmatter abstract" had to be interpreted as the Preface body. A grep for the `\paragraph{A note on formalism.}` pattern (which the review comment references) confirmed it lives in Chapter 1 (main matter), not in the frontmatter — so the correct update target was the Preface.

2. **Label validation regex scope**: LaTeX technically allows a broader set of characters in label keys (including `@`, `!`, etc. with certain packages), but restricting to `[A-Za-z0-9:_\-\.]` covers all label conventions actually used in the manuscript and excludes every character that `_tex_escape()` would mangle, making the validation conservative and safe.

3. **Venv bootstrap idempotency**: The existing `venv` target already checks for `$(VENV_PYTHON)` existence before creating the environment, so adding `venv` as a prerequisite to `index` does not add meaningful overhead on machines where the venv already exists.

## Next Ideas (6 Ideas)

2. **Add a `make check-index` target**: Run `generate_index.py` in dry-run / validation-only mode (no file write) and exit non-zero on any validation failure, so CI can block merges that would introduce broken cross-references.

3. **Pull the empirical-paragraph wording into a shared macro**: Define a `\newcommand{\empiricalsummary}{...}` in the preamble so the Preface and any future back-matter summary page stay in sync automatically.

4. **Typecheck `generate_index.py` with mypy in CI**: The new `str | None` return type annotation on `_validate_label_key` is already PEP 604 compatible; adding a `mypy --strict` pass to `make check` would catch future type regressions in the generator.

5. **Scotus-audit target portability**: `scotus-audit` still calls bare `python3`; apply the same `$(VENV_PYTHON)` treatment so all Python invocations in the Makefile use a consistent, dependency-controlled interpreter.

6. **Auto-regenerate `empirical_index.tex` in pre-commit hook**: Add a git pre-commit hook that runs `make index` and stages `Paper/empirical_index.tex` if it changed, ensuring the index is never out of sync with the registry at commit time.
