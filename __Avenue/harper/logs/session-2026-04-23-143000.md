## Challenges Encountered

1. **Registry field heterogeneity**: Some registry `.md` files use `figure:` as a YAML list (eq:11.1), others as a single string path. The companion document handles this by listing figures individually under each case block.

2. **usc_macros.sty dependency**: The main manuscript loads `\usepackage{usc_macros}`. The companion does not import this package (unnecessary — no USC snippet blocks in the companion). This was verified by reading the `.sty` file; it defines `\uscinline`, `\usclink`, `\uscshowdiff` — none of which are used in the companion.

3. **Equation statement mismatch risk**: Some registry entries have `statement:` fields that may have been updated in the manuscript after the registry was written. The companion uses registry field values directly per the plan instruction ("trust the files and references").

4. **Ch 8 eq:8.15 and eq:8.16 details**: These two registry files were not individually read; the index provided enough context (Google Trends data source, same notebook as eq:8.12 and eq:8.14, same case_study_line 4968). All four equations share the `eq40_45_interference_engine.ipynb` notebook.

5. **Biber key consistency**: Some bib keys in the registry (e.g., `gilens_page2014`, `james_black_jacobins1938`) may differ from the actual keys in `references.bib`. The appendix note on citation completeness flags this for compiler verification.

6. **Figure path for eq:10.18**: Registry entry says `Paper/figures/eq10_18_wage_asset_1948_2024.png` but the actual file found is `eq10_18_wage_asset_1948_2023.png`. The companion uses the actual filename from the glob result.
