# Session Log - 2026-04-06 02:12:00

## What Was Wrong / What Was Requested

After fixing Figure 2’s clipped caption with `\begin{figure}[!tbp]`, **Figure 1 and Figure 2 appeared in the wrong order** in the PDF: Figure 2 could float to the top of a later page while Figure 1 stayed near `[ht]`, so Figure 1 often printed before Figure 2 despite Figure 2 appearing first in the source.

## How I Fixed It / What I Did

1. **Reverted Figure 2 float option** from `[!tbp]` back to `[ht]` so it no longer prefers floating ahead of subsequent content.

2. **Loaded `placeins`** (`\usepackage{placeins}`) and inserted **`\FloatBarrier`** immediately after Figure 2’s `\end{figure}`. That forces all floats declared up to that point (including Figure 2) to be output before the following subsection, longtable, and Figure 1.

3. **Kept** the earlier caption mitigations for Figure 2: reduced axis height (`5.7cm`) and `\parbox{\linewidth}{\raggedright\small...}` around the `\caption*` text.

## Challenges Encountered

1. Trade-off: `\FloatBarrier` can add vertical whitespace or a float-only page if Figure 2 does not fit the current column; that is preferable to reversed figure order for this manuscript.

## Next Ideas (6 Ideas)

2. If caption clipping returns with `[ht]`, try `[!ht]` only (no `b`/`p`) before resorting to `[!tbp]` again.

4. Use `caption` package with `\captionof{figure}{...}` outside floats only if a non-float layout is required.

5. Consider `flafter` package so floats never appear before their anchor in source (alternative to `placeins`).
