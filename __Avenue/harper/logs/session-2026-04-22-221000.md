## How I Fixed It / What I Did

1. **Wrote `Paper/scripts/relabel_equations.py`** — a single Python script with four responsibilities:
   - Parse YAML frontmatter from all `.md` registry files to build `{old_label: new_label}` pairs, skipping any where they are equal or `new_label` is absent
   - Sort substitution pairs by descending key length (longest-first) to prevent partial-match collisions (e.g., `eq:10` inside `eq:100`)
   - Apply plain-string find-and-replace for all four LaTeX reference contexts (`\label{}`, `\ref{}`, `\eqref{}`, `eq.~\ref{}`) to the manuscript
   - Update the `label:` field in each registry `.md` file to the new value, leaving `new_label` intact as the historical record

2. **Audited special-case appendix references**: confirmed via `Grep` that:
   - `eq:73` → `eq:12.5-haitian-theorem-nonkinetic` and `eq:74` → `eq:12.6-haitian-theorem-kinetic` match the registry entries exactly
   - `eq:40`–`eq:45` (Interference Engine) all have registry entries and appear in the manuscript as LaTeX references

3. **Dry-run**: `python3 Paper/scripts/relabel_equations.py --dry-run` — reported 136 pairs, 294 site counts (against unfrozen original text), 136 registry files to update. No conflicts.

4. **Applied**: `python3 Paper/scripts/relabel_equations.py` — 266 actual substitution sites modified in the manuscript, 136 registry `.md` files updated with new `label:` values.

## Challenges Encountered

1. **Dead code cleanup**: The initial script draft contained a regex-based `replace_in_manuscript` function alongside a plain-string `replace_in_manuscript_v2`. The regex version was unnecessarily complex (and buggy — `re.escape()` on replacement strings would have corrupted the output). Removed it before running.

2. **Dry-run vs. apply count discrepancy (294 vs. 266)**: The dry-run counts each label against the frozen original text; the apply counts against the progressively-modified text, so earlier substitutions reduce the count of later (shorter) labels that would otherwise match regions already replaced. This is the correct behavior — the apply count (266) is the authoritative figure.

3. **`make pdf` dependency on `empirical`**: The `pdf` Makefile target depends on `empirical`, which executes notebooks including one with a pre-existing assertion failure (`historical_shocks.json`: expected 4 shocks, got 8). Bypassed by calling `latexmk` directly — the PDF compile is independent of that data pipeline.

4. **Prose references not handled**: The manuscript contains a handful of plain-text equation references like `"This is eq:74 confirmed"` (no LaTeX command). These are outside the four-context scope defined in the plan and were intentionally left unchanged. They are narrative prose, not cross-reference machinery.
