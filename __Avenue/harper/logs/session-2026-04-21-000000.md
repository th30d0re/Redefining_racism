## What Was Wrong / What Was Requested

Five verification comments were submitted after thorough review of the codebase. Each described a concrete portability, consistency, or completeness issue introduced during the previous session's Tier 1 case study implementation.

**Comment 1 — Makefile portability:**  
`venv` target hardcoded `/opt/homebrew/bin/python3`, which only exists on Apple Silicon Macs with Homebrew. Any collaborator on Linux or Intel Mac would get a silent failure on `make venv` / `make empirical`.

**Comment 2 — Notebook figure save paths:**  
All four notebooks saved figures using bare relative paths (`../figures/...`) and created the directory with `os.makedirs('../figures', ...)`. Data loading already used `_here` (a `pathlib.Path` resolved relative to the script file), but figure saving was inconsistent — fragile if `nbconvert` was invoked from a non-`Paper/scripts/` working directory.

**Comment 3 — Registry file format inconsistency:**  
Seven new registry files (`eq05.md`–`eq27.md`) used a Markdown table format (`## Summary`, `## Key Numbers`, etc.). Existing T4 registry files (e.g., `eq_1_08_suppression_envelope.md`) use YAML frontmatter with machine-parseable fields like `status:`, `tier:`, `difficulty:`, `falsification:`, `notebook:`. Two schemas coexisted for the same purpose.

**Comment 4 — Figures not embedded in manuscript:**  
All four notebooks generated and saved PNGs to `Paper/figures/`. None of the four `\subsection*{Case Study: ...}` blocks in `Redefining_Racism.tex` contained `\includegraphics` commands, so the compiled PDF showed no visual evidence.

**Comment 5 — CS4 numerical discrepancies:**  
Case Study 4 manuscript text cited slightly different values than what the notebook actually computes (Black: 6.94 vs. 7.00; Native Am.: 6.47 vs. 6.55; Hispanic: 2.86 vs. 2.89; ratio: 3.06× vs. 3.01×). The registry file `eq27.md` used the manuscript (wrong) values.

## How I Fixed It / What I Did

### Comment 1 — Makefile
Added `SYSTEM_PYTHON ?= python3` variable at the top of the `VENV` block and replaced the hardcoded path:
```makefile
SYSTEM_PYTHON ?= python3
# ...
$(SYSTEM_PYTHON) -m venv $(VENV); \
```
Users can override: `make venv SYSTEM_PYTHON=python3.12`.

### Comment 2 — Notebook figure paths
