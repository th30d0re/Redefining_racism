## Follow-up (2026-04-23) — `FileNotFoundError` on `to_parquet`

**Symptom:** `FileNotFoundError` writing `../../Paper/data/spatial/merged_tract_panel_*.parquet` — parent path did not exist.

**Cause:** `REPO_ROOT = Path("../../")` resolves relative to Jupyter’s **process** `cwd`, which is usually the repo root, not `Paper/scripts/`. From the repo root, `../../` points *outside* the project, so `DATA_DIR` was wrong and the output directory was never created.

**Fix:** Replaced with `_resolve_repo_root()` that walks upward from `Path.cwd()` until it finds `Paper/Redefining_Racism.tex` or `Paper/data/`, then `DATA_DIR.mkdir(parents=True, exist_ok=True)` (and same for `FIG_DIR`).

**Follow-up (same day) — Moran `I=nan` with `p=0.001` and SLM `n=0`**

- **Cause:** (1) `esda.Moran` / `Moran_BV` return `I=nan` when the series is constant or all-missing (zero variance), but `p_sim` from permutations can still be a small value — misleading. (2) `run_spatial_regression` used listwise complete cases; when PPI/lead/ACS are almost all missing, the intersection is empty.

- **Fix:** Added `_finite_std` / `_impute_median` guards; return `p=nan` when `I` is not finite. Wrapped `Moran` in `np.errstate` to silence divide warnings. Replaced listwise deletion with per-column median imputation for SLM/SEM, `panel.reindex(columns=x_cols)` so every regressor column exists, skip regression only if PPI is 100% missing, and log how many tracts were imputed.
