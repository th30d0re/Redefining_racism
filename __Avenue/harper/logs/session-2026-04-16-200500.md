# Session Log - 2026-04-16 20:05:00

## What Was Wrong / What Was Requested

Git unified diffs in the PDF (around page 291, `\uscshowdiff` / statute evolution) ran past the text margin because `\verbatiminput` does not wrap long lines.

## How I Fixed It / What I Did

1. **Pre-wrapped** `Paper/usc_snippets/usc_diff_*.diff` with `fold -s -w 88` so no physical line exceeds the typeblock width for monospace verbatim.
2. **Normalized** Unicode punctuation in those snippets to ASCII (smart quotes, em/en dashes, section sign) so pdfLaTeX + verbatim stays reliable.
3. **`Paper/usc_macros.sty`**: kept `\verbatiminput` inside a breakable `tcolorbox`, added `fontupper=\ttfamily\footnotesize`, and kept a wrapping-friendly **title** (`\parbox{0.92\linewidth}{\raggedright\footnotesize #1}`) so long `Git diff: \texttt{...}` titles do not run off the page.
4. **`tools/usc_write_diffs.sh`**: extended `sanitize_diff` with the same Unicode-to-ASCII map and added `wrap_diff_for_pdf` (`fold -s -w 88`) after each written diff so `make usc-all` / preset regeneration stays PDF-safe.

**Note:** `listings` with `breaklines=true` on these OLRC diffs hit **TeX capacity exceeded** (known fragile interaction with long tokens); pre-wrapping the files avoids that class of failure.

## Challenges Encountered

1. `listings` + `breaklines=true` exhausted main memory on long statute lines.
2. `fvextra` + `breaklines` inside a breakable `tcolorbox` also exhausted memory.
3. `listingsutf8` / UTF-8 paths still tripped “Invalid UTF-8” until snippets were ASCII-normalized.
